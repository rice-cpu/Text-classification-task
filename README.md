# 基于BERT的今日头条新闻文本分类系统 (Demo 1)

## 项目简介
本项目是基于 `bert-base-chinese` 预训练模型实现的中文新闻文本分类任务。
---

## 环境与核心依赖
运行本项目需要确保本地已安装以下标准库（建议在 PyCharm Terminal 中一键安装）本项目已做安全重构。运行前请在根目录下自行新建 config.json 文件，并参照以下格式配置您的 SwanLab 密钥及超参数：
- Python 3.8+
- PyTorch (建议配置好 GPU 显卡车间)
- Transformers (Hugging Face 词表与模型库)
- Scikit-learn (用于计算最终 Accuracy 准确率)
- SwanLab (用于全流程训练轨迹可视化)

---

## 数据集与密码本对账
- **数据来源**：今日头条文本分类数据集（采用训练集 `train_3k.txt` 3000条，验证集 `dev_1k.txt` 1000条）。
- **数据清洗**：手工编写 `parts = line.split("_!_")` 线性流过滤损坏数据。
- **密码本映射**：代码会自动扫描训练集和验证集的所有唯一标签，并动态组装出数字映射密码本（`label_map`），确保 CrossEntropyLoss 交叉熵损失函数顺利对账。

**注意**：运行前无需改动任何 `.py` 源代码。请务必前往本地的 `config.json` 账本中，将 `"data_dir"` 修改为你本地存放数据集的真实文件夹路径（支持相对路径或绝对路径）。
---

## 核心调参迭代与优化心路历程

本项目一共经历了**三轮硬核迭代**，最终完美突破 83% 的硬性达标线：

### 1. 阶段一：过拟合(Batch_Size=4)
最开始由于漏掉了 Epoch 循环最前端的 `model.train()`，导致 BERT 内部的 Dropout 保安集体下班。模型在第 3 轮后陷入死记硬背，验证集准确率断崖式暴跌至 **77.8%**。

### 2. 阶段二：学术规范化与引入调度器 (Batch_Size=16)
做出针对性修正：
- 将 `BATCH_SIZE` 扩大至 16（平滑梯度方向，减少噪声）；
- 在循环内部首行锁死 `model.train()` 和 `epoch_loss = 0.0`；
- 引入 `CosineAnnealingLR`（余弦退火学习率调度器）。
此时模型表现极其平稳，不低头、不暴跌，稳定在 **83.0%** 达标线附近。但由于最后阶段学习率过早退火枯竭，第 8、9 轮出现无法突破的躺平直线。

### 3. 阶段三：主矛盾突破与拉长倒计时 (T_max=15, max_length=64)
- **语义长度设置**：发现最初设定的 `max_length=32` 会导致长标题新闻在末尾被阶段，丢失了决定分类的核动词（如“收窄”等）。将其**强行扩展至 64** 确保文本语义完整！

### 4.阶段四：规范重构与算法防御迭代

在项目顺利跑通 baseline 后，为了满足工业级生产环境的安全规范与高可用要求，本项目在师兄团队的深度评审与指导下，开展了全方位的底层架构解耦与算法缺陷防御重构。
本阶段的核心优化对账单如下：

#### (1). 核心资产隔离与安全防线（.gitignore 熔断机制）
* **痛点问题**：旧版本缺乏统一的资产保护，大文件模型权重（`checkpoints/`）、本地开发缓存（`.idea/`）以及包含 SwanLab/DeepSeek 私密密钥的明文账本（`config.json`）存在误传至公开 GitHub 仓库的重大安全隐患。
* **重构方案**：在项目根目录强制引入工业标准 `.gitignore` 熔断机制，建立绝对资产黑名单，实现敏感文件与胖权重的“云端隐形”，将核心密码泄露风险彻底降为 0。

#### (2).彻底的数据流与模型解耦（Decoupling Architecture）
* **痛点问题**：旧版的 `dataset.py` 自行跑去硬盘进货、清洗数据，不仅导致 Dataset 仓库结构臃肿，而且在更换数据集或调整超参时，极易引发越界崩溃。
* **重构方案**：将数据读取（`with open`）、不规则行清洗（`if len(parts) < 4: continue`）全部放入 `main.py` 。数据清洗成标准列表后，再给 `testDataset`。实现了“数据读取”与“模型训练”的彻底解耦。

#### (3).标签安全映射与对账补丁（Label Mapping）
* **痛点问题**：原始数据集的标签为不连续 ID（如 102、104），而 PyTorch 底层 `nn.CrossEntropyLoss` 死死规定类别标签必须是从 0 开始的连续整数。强行喂入会导致显卡原地爆炸（`CUDA error: device-side assert triggered`）。
* **重构方案**：在数据入场处，通过 `set(all_raw_labels)` 实时建立全局大厂规范的数字标签连续映射账本（如将 102、104 自动映射为 0、1），全自动对账。模型分类器通道与真实类别数完美动态对齐（共有 15 个类别），彻底根治了显卡崩溃隐患。

#### （4）. 特征遮罩与分层差学习率（Attention Mask & Layer-wise differential learning rate）
* **痛点问题**：旧版本未引入 `attention_mask`，导致模型会偷偷通过数无意义的 `[PAD]` 零元素“走捷径作弊”来强行提高Acc，暴露了严重的过拟合与特征泄露问题。
* **重构方案**：
  * **特征排毒**：强行焊上 `attention_mask` 遮罩，强制将所有 `[PAD]` 权重归零，抹除作弊痕迹，逼迫BERT靠硬实力攻克真实的 15 类新闻汉字。
  * **分层差学习率**：在排除作弊水分后，通过将 `max_length` 拓宽至128以倍增信息视野，并引入**分层差分学习率**（给 BERT 预训练身躯 $5\times10^{-6}$ 的温柔微调速度，给全新分类头 $5\times10^{-4}$ 的高速鞭策），配合余弦退火（CosineAnnealingLR）在有限的 3k 限定数据天花板下，成功压榨出模型真正的底座实力！

---
**重构阶段成果**：本轮重构虽然经历了“排除作弊虚高水分”带来的 Acc 真实震荡，但在SwanLab云端看板的实时对账下，模型走向了真正干净、透明、举一反三的学习曲线，最优权重已完美封印于本地 `checkpoints/best_model.pth` 中。
**最终战果**：最优 Checkpoint 自动触发 `torch.save` 机制，**成功斩获 84.10% 的泛化准确率**！最优权重已稳稳锁死在 `./checkpoints/best_model.pth` 中。
